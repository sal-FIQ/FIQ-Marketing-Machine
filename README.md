# Marketing Machine Repository

This repository is a starter kit for building a modular “marketing machine” that takes advantage of foundational context files. It is inspired by the approach described by Kieran Flanagan and adapted for use with Claude Code or other LLM‑powered systems.

The core idea is to centralize the core intelligence (audience insights, tone of voice, positioning, and buyer journey) in a handful of reusable Markdown files. Skills can then dynamically load only the portions of that context that apply to a given task.

## Structure

```
marketing-machine/
├─ foundation/
│  ├─ audience-delight.md
│  ├─ creator-style.md
│  ├─ customer-journey.md
│  └─ market-positioning.md
├─ prompts/
│  └─ reusable-instructions.md
├─ skills/
│  └─ (empty for now – add your skill folders here)
├─ content/
│  └─ (place generated outputs or work products here)
├─ research/
│  └─ (use this to store transcripts, notes, or source material)
└─ README.md
```

### `foundation/`

The four foundation files encapsulate your organization’s core knowledge:

* **audience‑delight.md** — how your audience talks, what delights them, and the language they use when no one from your company is listening.
* **creator‑style.md** — your authentic voice and tone, extracted from real content.
* **customer‑journey.md** — the actual discovery, evaluation, and conversion moments your buyers go through.
* **market‑positioning.md** — your strategic claim, competitors’ claims, and the current market narratives.

Each file begins with a header that specifies when to load it. Skills should read only the header first to decide whether the file is relevant before loading the full contents.

For a table summarizing each foundation file and tips on maintaining them, see `foundation/README.md`.

### `prompts/`

This directory contains reusable prompt blocks. The file `reusable‑instructions.md` describes a standard three‑step pattern for loading foundation context, executing task‑specific work, and self‑reviewing the result.

You can learn how to create additional prompt fragments in `prompts/README.md`.

### `skills/`

Add one subdirectory per skill. Inside each skill folder, include a `SKILL.md` (or other file) that describes how the skill operates. Start by importing the foundation loading instructions from `prompts/reusable‑instructions.md` or copying the pattern directly into each skill.

For example, the beginning of a skill definition might look like this:

```md
# LinkedIn Post Skill

## Instructions

### Step 1: Load Foundation Context

Scan `./foundation/` for all `.md` files. For each file:
1. Read only the header block above the first `---`.
2. Check whether the “Load this file when” criteria match this task.
3. Load the full file only if it is relevant.
4. Skip files that do not match.

Do not load every foundation file by default. Use only the minimum relevant context needed for this task.

### Step 2: Draft the post
…
```

When you are ready to create new skills, simply add subdirectories under `skills/` and follow this pattern.

See `skills/README.md` for a step‑by‑step guide to creating and organizing your skills.

### `content/`

This folder is where you store the **outputs generated by your skills**—blog posts, social updates, emails, landing pages, and other deliverables. Organize the files by type or date to keep drafts and final copies tidy. More guidance on managing deliverables can be found in `content/README.md`.

### `research/`

The `research` directory holds all **raw source materials** such as transcripts, surveys, competitive analyses and industry reports. Keep your research organized so you can easily update your foundation files as new insights emerge. For detailed suggestions on organizing research, refer to `research/README.md`.

## Using This Repository

1. **Load only what you need.** Each skill starts by scanning the `foundation/` folder and loading only the files whose header criteria match the current task. This prevents overload and ensures focused outputs.
2. **Keep foundations up to date.** When your audience, voice, positioning or buyer journey evolve, update the corresponding foundation file. All skills will benefit automatically.
3. **Add new context thoughtfully.** If you find yourself copying the same context into multiple skills, consider whether it belongs in a new foundation file.

Feel free to extend this structure with additional directories (e.g., `inputs/`, `outputs/`) to suit your workflow. The key is to separate reusable intelligence from task‑specific instructions and let your skills decide what context they need at runtime.
