# Content Directory

Use the `content` folder to store **outputs generated by your skills**. These files are the final deliverables that you publish or share with stakeholders—blog posts, social media updates, email sequences, landing pages, ad copy, and more.

## Organizing Outputs

Consider organizing your content by type or date, for example:

```
content/
├─ blog-posts/
│  ├─ 2026-04-15-launch-recap.md
│  └─ 2026-05-01-ai-productivity-guide.md
├─ linkedin-posts/
│  ├─ 2026-04-20-new-feature-announcement.md
│  └─ 2026-04-27-customer-story.md
└─ email-campaigns/
   ├─ nurture-sequence-01/
   │  ├─ email-1.md
   │  ├─ email-2.md
   │  └─ email-3.md
   └─ launch-messaging-2026-05.md
```

Keeping your outputs organized makes it easier to track revisions, share drafts for feedback, and reuse successful content patterns.

## Best Practices

* Include metadata such as creation date, author, and context where relevant.
* Store only final or near‑final drafts here; use the `research/` folder or a separate working directory for messy notes and early iterations.
* If the content is derived from a specific skill and set of foundation files, mention that at the top of the file for traceability.
